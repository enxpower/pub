# AGENTS.md — enxpower/pub 发布治理

本文件是本仓库内所有 AI、开发者和自动化工具的最高执行入口。

## 1. 仓库定位

`enxpower/pub` 是 Andy 用于协助朋友、合作伙伴和个人项目发布独立 HTML 内容的公共发布仓库。

- 自定义域名：`https://enxpower.com`
- 每个根目录子文件夹对应一个独立发布项目。
- 发布地址：`https://enxpower.com/<folder>/`
- URL **不得**包含仓库名 `/pub/`。
- 每个项目必须相互独立，不得修改其他项目文件。

## 2. 新项目默认任务

当用户在新线程提供素材，并表达“发布到 pub”“按发布库规范制作”“建立新文件夹和 index.html”等意图时，立即执行：

1. 阅读本文件。
2. 阅读 `docs/PUBLISHING_GOVERNANCE.md`。
3. 阅读 `docs/HTML_PRODUCT_STANDARD.md`。
4. 核对仓库当前 `main` 分支真实状态。
5. 在仓库根目录创建一个语义明确、全小写、短横线分隔的新文件夹。
6. 在该文件夹内创建完整、可直接访问的 `index.html`。
7. 创建 PNG 社交分享图、favicon 及项目所需本地资源。
8. 完成电脑、Pad、手机三端核验及全部链接/按钮/锚点检查。
9. 提交到 `main` 并核验正式线上 URL。
10. 最终向用户提供正式访问地址、源码位置和本次提交 SHA。

除非存在真正阻塞执行的事实性缺失，否则不要反复询问，不要只给方案，不要停留在草稿。

## 3. 内容权威顺序

1. 用户本轮明确要求
2. 用户提供的原文、素材和事实资料
3. `docs/PUBLISHING_GOVERNANCE.md`
4. `docs/HTML_PRODUCT_STANDARD.md`
5. 既有项目文件

不得为了视觉效果篡改原文核心思想，不得虚构合作关系、成果、数据、授权或实施状态。

## 4. 目录规则

标准结构：

```text
<project-slug>/
├── index.html          # 唯一正式入口，必须是完整静态 HTML
├── og-image.png        # 标准社交分享图，优先 1200×630 PNG
├── wechat-share.png    # 需要时提供微信方图，建议 500×500 PNG
├── favicon.png         # 可选；也可使用内嵌 favicon
└── assets/             # 仅该项目使用的图片、字体以外资源
```

禁止：

- 把新项目直接写进仓库根 `index.html`
- 使用 JavaScript 跳转壳替代完整 `index.html`
- 依赖构建后才能生成正文，但仓库中没有稳定构建流程
- 引用本机路径、临时沙箱路径或不可公开访问的资源
- 修改无关项目
- 把字体文件提交或对外分发

## 5. 发布铁律摘要

所有项目必须：

- 精确适配电脑、Pad、手机。
- 全端锁死横向滚动。
- 手机端在需要导航时使用可操作的汉堡菜单。
- `index.html` 直接包含完整页面，不依赖 JS 二次加载正文。
- 配置与内容强相关的标题、简介、favicon、OG/Twitter 元数据。
- 分享图必须使用公开可访问的 PNG；微信兼容性优先。
- 所有按钮、导航、CTA、锚点发布前逐一验证。
- 所有装饰层不得拦截触摸和点击，必要时设置 `pointer-events:none`。
- 使用对应主体 VI；无明确企业主体时使用 GONG-VI。
- 页面精美、克制、易读，不做在线 Word，不堆砌内容。
- 不出现 TODO、占位符、内部沟通术语、AI 生成痕迹或未经确认的状态。
- 不提供“邮件查看源代码”等暴露源码的设计。

完整标准以 `docs/HTML_PRODUCT_STANDARD.md` 为准。

## 6. Git 与发布规则

- 默认基于当前 `main` 开发并直接提交到 `main`，除非用户明确要求 PR 流程。
- 写入前必须核对目标路径不存在或读取现有文件 SHA。
- 不得声称已经发布，除非文件已真实写入仓库。
- 不得猜测正式 URL。自定义域名下的地址是：

```text
https://enxpower.com/<folder>/
```

GitHub Pages 备用地址通常是：

```text
https://enxpower.github.io/pub/<folder>/
```

正式对外传播优先使用 `enxpower.com`。

## 7. 发布前最低验收

必须核验：

- [ ] `index.html` 存在并可直接渲染完整内容
- [ ] 页面标题、简介与项目内容一致
- [ ] 所有 `href="#..."` 均有对应 `id`
- [ ] 所有按钮可点击，移动端无透明覆盖层
- [ ] 手机、Pad、桌面无横向滚动
- [ ] 图片无 404，资源路径区分大小写
- [ ] OG 图片为 PNG 且使用绝对 HTTPS URL
- [ ] 微信分享候选图真实存在、MIME 正确、尺寸合理
- [ ] favicon 有效
- [ ] 无虚构数据、无错误合作状态、无内部占位文字
- [ ] 正式地址不包含 `/pub/`

## 8. 标准一句话启动口令

用户在任何新线程只需说：

> 使用 `enxpower/pub` 发布库，按仓库 `AGENTS.md` 和 HTML 铁律，把我提供的素材制作成独立精美中文网页；在根目录创建新文件夹和完整 `index.html`，完成三端适配、PNG 分享预览、favicon、按钮与锚点核验，直接发布并给我正式链接。

收到这句话后，应立即读取仓库治理文件并开工。
