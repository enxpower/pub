# 新线程一句话启动口令

以后在任何新线程中，先上传或粘贴素材，然后直接说：

> 使用 `enxpower/pub` 发布库，按仓库 `AGENTS.md` 和 HTML 铁律，把我提供的素材制作成独立精美中文网页；在根目录创建新文件夹和完整 `index.html`，完成三端适配、PNG 分享预览、favicon、按钮与锚点核验，直接发布并给我正式链接。

这句话已经包含以下默认授权和要求：

- 读取 `enxpower/pub` 当前真实状态
- 遵守仓库治理文件
- 在仓库根目录建立独立项目文件夹
- 使用稳定英文 slug
- 创建完整静态 `index.html`
- 忠实表达原文核心思想
- 以高水平设计和传播逻辑重构内容
- 适配电脑、Pad、手机
- 锁死横向滚动
- 配置手机汉堡菜单（需要时）
- 创建 PNG 社交分享图和 favicon
- 核验微信分享链路
- 核验所有按钮、CTA 和锚点
- 直接提交并发布
- 返回正式 `enxpower.com/<folder>/` 地址和 commit SHA

## 可选补充方式

需要指定项目名时：

> 文件夹名使用 `<project-slug>`，其他全部按 `enxpower/pub` 的 AGENTS.md 和 HTML 铁律直接完成并发布。

需要指定语言时：

> 先做纯中文，其他全部按 `enxpower/pub` 的 AGENTS.md 和 HTML 铁律直接完成并发布。

只允许局部修改时：

> 只修改我指出的问题，页面正文和既有设计不要动；按仓库治理做最小差异修复并全面核查同类问题。

## AI 收到后应立即做什么

1. 读取 `AGENTS.md`
2. 读取 `docs/PUBLISHING_GOVERNANCE.md`
3. 读取 `docs/HTML_PRODUCT_STANDARD.md`
4. 核对 `main` 分支和目标目录
5. 阅读全部素材
6. 设计、开发、验证、提交、发布

不应先要求用户重复仓库规则，也不应只输出设计建议而不落地。
